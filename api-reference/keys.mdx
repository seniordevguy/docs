# API Keys

API keys provide programmatic access to the Recall API. Keys are scoped to the user who creates them and inherit that user's organization and permissions.

## Create a Key

<Card>
  `POST /api/keys`
</Card>

```bash
curl -X POST https://recall.yourdomain.com/api/keys \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer rk_your_api_key_here" \
  -d '{
    "name": "CI/CD Pipeline",
    "scopes": ["chat"],
    "expiresInDays": 90
  }'
```

<ParamField body="name" type="string" required>
  A descriptive name for the key.
</ParamField>

<ParamField body="scopes" type="array" default={[]}>
  Array of scope strings to restrict the key's access. An empty array grants full access.
</ParamField>

<ParamField body="expiresInDays" type="number">
  Number of days until the key expires. Omit for a non-expiring key.
</ParamField>

### Response

```json
{
  "key": "rk_live_a1b2c3d4e5f6...",
  "record": {
    "id": "key_abc123",
    "name": "CI/CD Pipeline",
    "scopes": ["chat"],
    "createdAt": "2025-02-25T01:00:00Z",
    "expiresAt": "2025-05-26T01:00:00Z"
  }
}
```

<Warning>
  The `key` field contains the full API key and is **only returned once**. Store it in a secure location (e.g. environment variable, secrets manager). It cannot be retrieved again.
</Warning>

## List Keys

<Card>
  `GET /api/keys`
</Card>

```bash
curl https://recall.yourdomain.com/api/keys \
  -H "Authorization: Bearer rk_your_api_key_here"
```

<ResponseField name="keys" type="array">
  Array of key records for the authenticated user. Key values are **not** included — only metadata (id, name, scopes, createdAt, expiresAt, lastUsedAt).
</ResponseField>

## Revoke a Key

<Card>
  `DELETE /api/keys/{id}`
</Card>

```bash
curl -X DELETE https://recall.yourdomain.com/api/keys/key_abc123 \
  -H "Authorization: Bearer rk_your_api_key_here"
```

Immediately invalidates the key. Returns `{"ok": true}` on success or `404` if the key doesn't exist or doesn't belong to the authenticated user.
