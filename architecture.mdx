# Architecture

Recall is a monolithic Next.js 15 application backed by PostgreSQL with pgvector for semantic search. It uses Docker containers for sandboxed code execution and browser automation, and streams AI responses via Server-Sent Events (SSE) using the Vercel AI SDK.

## Core Components

### Chat Engine

The chat engine handles streaming AI responses with multi-step tool calling. Each conversation turn can execute up to **25 autonomous tool steps** вҖ” the AI can search the web, execute code, browse pages, and chain results together without user intervention.

- **Provider resolution**: User key вҶ’ Org key вҶ’ System fallback (OpenRouter)
- **Streaming**: Vercel AI SDK data streams over SSE
- **Auto-compaction**: Long conversations are automatically summarized to fit within model context windows
- **Auto-titling**: First message generates a conversation title via GPT-4o-mini

### Knowledge System

Knowledge entries are injected into conversations using a two-phase approach:

1. **Always-inject**: Entries with `priority: "always"` are included in every request (instructions, policies)
2. **Semantic retrieval**: Entries with `priority: "relevant"` are matched against the user's message using pgvector cosine similarity with a **0.35 threshold**

Knowledge is scoped hierarchically: **org вҶ’ group вҶ’ user**. All applicable scopes are merged at query time.

### Skill Framework

Skills are the tool-use layer. Each skill is registered in the database and gated via `skillReady()` вҖ” a skill is only available if it's installed, configured, and has valid credentials. There are **no environment variable fallbacks**; everything goes through the DB.

Skills include:
- **Web Search** (Brave API)
- **Code Execution** (Python/Node.js in persistent Docker containers)
- **Browser Automation** (Playwright in Docker)
- **Image Generation** (DALL-E and compatible APIs)
- **OAuth Integrations** (Google Workspace, GitHub, Microsoft 365, Slack)
- **Weather**, **Memory**, **File I/O**, and more

### Authentication

Recall uses [Better Auth](https://better-auth.com) for authentication:
- **Google SSO** вҖ” OAuth sign-in
- **Email/Password** вҖ” Credential-based authentication
- **API Keys** вҖ” `rk_` prefixed keys for programmatic access, hashed at rest

### Database

PostgreSQL with the **pgvector** extension provides:
- Relational storage for users, orgs, conversations, messages, knowledge, skills
- Vector embeddings for semantic knowledge retrieval
- Full-text search across conversation content

### Docker Sandboxing

Code execution and browser automation run in isolated Docker containers:
- **Persistent containers per conversation** вҖ” state (files, variables) persists across tool calls within the same conversation
- **Network access** вҖ” containers can fetch external resources
- **Resource limits** вҖ” CPU and memory constraints prevent abuse
- **Automatic cleanup** вҖ” containers are removed when conversations are inactive
